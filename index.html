Community Server Services in Townships

Project Description

Township and rural areas usually lack the infrastructure needed to have reliable and cost effective internet connectivity [1]. Internet connections in townships are typically slow, low bandwidth wireless connections when compared to internet connections in developed regions [2]. With the emergence of Web 2.0 technologies, the Web has become more socially connected and people are able to share content via the Web more often than before [5]. It has been noted that there is a lack of local content on the Web from rural Africa and that the Internet is often perceived as a “foreign body” [12]. The importance of low cost internet connectivity in township and rural areas is motivated by the benefits towards education and the overall development of the community [3]. Internet performance and cost is an obstacle that prevents local community members from sharing content via the Web. Additionally, this prevents usage of high-bandwidth Web applications such as Skype and YouTube.

(Kim, 2013)

Problem Statement

Poor communities in rural areas and townships lack fast and efficient internet connectivity at an affordable cost. For example the Internet connection in Macha, a rural settlement in Zambia is provided through a satellite gateway that has a committed download speed of 128 kbps bursting to 1 Mbps which is below the recommended minimum of 4mbps [5]. The total monthly cost of the connection is $1200 (USD) which is unaffordable to the low-income rural residents. With evidence of strong locality of interest in these communities as evidenced in Macha Zambia, a fast, efficient and free local network for file sharing and communication can be set using open source tools [5]. The research questions below aim to discover how this local cloud can be improved to provide a better service to its users in a community.

Research Questions:

1. Can an intelligent decision tree synchronisation technique reduce internet data costs as opposed to a simplistic first in first out (FIFO) synchronisation technique, and how does data availability of the decision tree technique compare to the first in first out (FIFO) technique, for users that travel to different local servers?

2. Can a peer-to-peer file sharing method be used as a fall back solution in the event of the server being unavailable and still maintain one hundred percent availability of data, and how does the average bandwidth of the peer-to-peer connections compare to the average bandwidth of the client-cloud connection?

3. Can a decentralized social network be integrated with other server applications to increase the usage and ease of access (through social network login) of server applications?
